Test voor Ghost Connection Bug Fix

Beschrijving van de Bug

Bij het plaatsen van een tussennode verdwijnt de originele verbinding, maar komt deze visueel terug wanneer:

  • Een van de nieuwe verbindingen wordt geselecteerd
  • Een "ghost" verbinding verschijnt tussen de hoofdnode en de nieuwe node (niet de tussennode)
  • Bij klikken op deze ghost verbinding verschijnt er een nieuwe verbindingslijn

Testprocedure

Stap 1: Open index.html in je browser
Stap 2: Maak twee nodes en verbind ze
Stap 3: Klik op het + icoon op de verbinding om een tussennode toe te voegen
Stap 4: Selecteer een van de nieuwe verbindingen door erop te klikken
Verwacht resultaat: De geselecteerde verbinding wordt blauw, geen ghost verbindingen verschijnen
Bug resultaat (voor fix): Een ghost verbinding verschijnt tussen de oorspronkelijke nodes

Wat is er gefixed?

  • rendering.js regel 38: refreshConnections() vervangen door drawConnection(connection)
  • rendering.js regel 419: refreshConnections() vervangen door drawConnection(connection)
  • utils.js regel 148: Dubbele refreshConnections() verwijderd
  • rendering.js: Extra check toegevoegd of verbinding nog bestaat in connections array
  • rendering.js: Oude DOM elementen proper opruimen voordat nieuwe worden toegevoegd

Console Debug Commands

Gebruik deze commands in de browser console om de status te checken:

console.log('Aantal verbindingen:', connections.length);
console.log('Verbindingen:', connections.map(c => ({ id: c.id, source: c.source, target: c.target })));
console.log('DOM verbindingen:', document.querySelectorAll('.connection').length);
